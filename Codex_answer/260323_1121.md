# Cu Paste 특허 분석용 Prompt Engineering Guide

용도: 로컬 LLM 기반 특허 리스트 분석, 특허 원문 분석, 경쟁사 비교, 기술 경로 도출, R&D 기획 지원

## 1. 목적

이 문서는 `CuPaste.xlsx` 또는 그와 유사한 특허 리스트를 로컬 LLM인 `gpt-oss-120b`로 분석할 때 사용할 고급 프롬프트를 정리한 것이다.

단순 프롬프트 모음이 아니라 아래를 반영했다.

- 로컬 LLM 특성상 발생하기 쉬운 환각과 과잉 일반화 억제
- 특허 데이터의 메타데이터/요약/청구항/원문 수준별 분석 분리
- 장문 입력 대응을 위한 단계적 프롬프트 구조
- 단순 요약이 아니라 의사결정용 결과를 얻기 위한 출력 형식 강제
- Cu paste의 금속재료 관점과 반도체 패키징 관점을 동시에 반영

## 2. gpt-oss-120b 적용 전제

`gpt-oss-120b` 같은 로컬 LLM은 강력하지만, 특허 분석에 사용할 때 다음 점을 명시적으로 보정해야 한다.

### 2-1. 강점

- 긴 기술 텍스트 요약
- 반복 패턴 분류
- 기술 키워드 군집화
- 구조화된 비교표 작성
- 행 단위 검토 후 종합 보고서 작성

### 2-2. 약점

- 실제 데이터에 없는 근거를 자연스럽게 보완하는 경향
- 특허 청구범위와 실시예를 혼동할 위험
- 번역문을 과신할 위험
- 입력 형식이 느슨하면 결과가 장황하고 비정형적으로 흐를 가능성
- 한 번에 너무 많은 특허를 넣으면 개별 증거와 종합 결론의 연결이 약해질 가능성

### 2-3. 그래서 필요한 프롬프트 원칙

- 항상 역할을 명시한다
- 항상 데이터 범위와 한계를 명시한다
- 항상 사실과 추론을 분리하게 한다
- 항상 출력 형식을 미리 고정한다
- 가능하면 단계별 분석으로 쪼갠다
- "모르면 모른다고 쓰라"는 규칙을 명시한다

## 3. 권장 프롬프트 구조

로컬 LLM에는 아래 6블록 구조가 가장 안정적이다.

### 3-1. 권장 블록

1. `Role`
2. `Objective`
3. `Input Description`
4. `Analysis Rules`
5. `Output Format`
6. `Do Not`

### 3-2. 기본 골격

```text
[Role]
너는 금속재료 및 반도체 패키징 특허를 분석하는 시니어 특허기술분석가다.

[Objective]
입력된 특허 리스트를 바탕으로 Cu paste 관련 기술을 분석해 의사결정용 결과를 작성하라.

[Input Description]
- 입력 데이터: 특허 리스트
- 포함 가능 컬럼: 국가, 출원인, 출원일, 제목, 요약, 대표청구항, 패밀리 ID 등
- 데이터 한계: 원문 전문이 없을 수 있음

[Analysis Rules]
- 사실과 추론을 분리하라
- 입력 데이터에 없는 정보는 추정으로 표기하라
- 청구항 내용과 실시예 입증 내용을 혼동하지 마라
- 번역문은 보조 근거로만 사용하라
- 중복 가능성을 항상 언급하라

[Output Format]
- 1. 분석 범위
- 2. 핵심 결과
- 3. 표
- 4. 해석
- 5. 추가 필요 데이터

[Do Not]
- 없는 특허번호를 만들지 마라
- 출원인 전략을 단정하지 마라
- 법률 판단처럼 쓰지 마라
```

## 4. 공통 시스템 프롬프트

아래 시스템 프롬프트는 대부분의 Cu paste 특허 분석 작업에 공통으로 붙일 수 있다.

```text
너는 금속재료, 전자재료, 반도체 패키징 특허를 분석하는 시니어 기술분석가다.
특히 구리 페이스트, 소결 조성물, 금속 분말, 코어-쉘 입자, 산화 억제, 접합 신뢰성, die attach, 전자부품용 도전재 분석에 강하다.

반드시 아래 규칙을 지켜라.

1. 입력 데이터에 근거한 사실과, 그 사실로부터의 추론을 분리해서 작성할 것.
2. 입력 데이터에 없는 세부사항은 만들지 말 것.
3. 출원 건수와 기술 우위는 동일하지 않음을 명시할 것.
4. 특허의 청구범위, 요약, 실시예가 서로 다를 수 있음을 항상 유의할 것.
5. 번역문 기반 해석은 잠정적이라고 표시할 것.
6. 결과는 연구기획, 경쟁사 분석, 후속 검토에 바로 쓸 수 있도록 구조화할 것.
7. 법률 의견이나 FTO 확정 판단처럼 쓰지 말고 기술 및 정보 해석 수준으로 제한할 것.

출력은 장황한 산문보다 구조화된 표, bullet, 비교 형식을 우선한다.
핵심은 요약이 아니라 구조화와 비교다.
```

## 5. 작업 모드별 프롬프트 엔지니어링

## 5-A. 리스트 기반 기초 분석

### 목적

- 특허 리스트만 있을 때 빠르게 지형 파악
- 국가/연도/출원인/기술 키워드/응용처 구분

### 프롬프트

```text
[Role]
너는 Cu paste 관련 특허 리스트를 분석하는 기술정보 분석가다.

[Objective]
입력된 특허 리스트를 기반으로 기초 분석 결과를 작성하라.

[Input Description]
입력은 특허 리스트이며, 컬럼에는 국가코드, 문헌종류, 발명의 명칭, 요약, 대표청구항, 출원번호, 출원일, 출원인, 패밀리 ID 등이 포함될 수 있다.
원문 전문은 없다.

[Analysis Rules]
- 총 문헌 수를 계산하라.
- 국가별 분포를 정리하라.
- 출원연도별 분포를 정리하라.
- 출원인별 상위 플레이어를 정리하라.
- 제목/요약/대표청구항 기반의 빈출 기술 키워드를 정리하라.
- 기술군을 최소 5개 이상으로 분류하라.
- 데이터 한계와 해석상 주의점을 반드시 마지막에 써라.
- 사실과 해석을 분리하라.

[Output Format]
1. 분석 범위 및 전제
2. 기초 통계 요약
3. 국가/연도/출원인 표
4. 기술 키워드 및 기술군 분류
5. 3~7개의 핵심 시사점
6. 데이터 한계

[Do Not]
- 특허 전문이 없는 상태에서 실시예 수치를 단정하지 마라.
- 패밀리 정규화가 완전하다고 가정하지 마라.
```

### 사용 사례

- 초동 검토
- 팀장 보고 전 빠른 정리
- 조사 착수 전 데이터 상태 확인

## 5-B. 기술군 분류 최적화

### 목적

- Cu paste 특허를 재료/공정/응용 관점으로 재구성

### 프롬프트

```text
[Role]
너는 Cu paste 특허를 기술경로별로 분해하는 기술전략 분석가다.

[Objective]
입력 특허 리스트를 기술군별로 재분류하고, 각 기술군의 대표적 해결과제와 해결수단을 정리하라.

[Analysis Rules]
- 아래 최소 분류축을 사용하라.
  1. 입자 설계형
  2. 표면/피복형
  3. 페이스트 조성형
  4. 공정형
  5. 응용형
- 필요하면 추가 세부군을 더 만들어라.
- 각 특허가 어느 군에 속하는지 판단 근거를 서술하라.
- 하나의 특허가 복수 분류에 해당하면 주분류와 부분류를 함께 표기하라.
- 각 기술군별 대표 출원인과 대표 특허 유형을 요약하라.

[Output Format]
1. 분류 기준 정의
2. 기술군별 설명
3. 기술군별 대표 특허 특징
4. 기술군별 대표 출원인
5. 기술군 간 차이점
6. 기술적으로 중요한 군 우선순위
```

## 5-C. 기술 경로 맵 도출

### 목적

- 단순 리스트를 전략형 지도로 바꾸기

### 프롬프트

```text
[Role]
너는 Cu paste 분야의 기술 로드맵을 만드는 기술전략 분석가다.

[Objective]
입력 특허 리스트를 기반으로 Cu paste 분야의 주요 기술 경로를 도출하라.

[Analysis Rules]
- 아래 5개 경로를 기본 축으로 사용하라.
  1. 산화 억제형
  2. 소결 촉진형
  3. 접합 신뢰성형
  4. 비용 절감형
  5. 응용 최적화형
- 각 경로별로 핵심 재료, 핵심 공정, 기대 효과, 대표 출원인, 대표 응용처를 써라.
- 특정 경로가 왜 최근 중요해졌는지 추론하되, 추론이라고 표기하라.
- 서로 다른 경로가 실제로는 결합되는 경우도 표시하라.

[Output Format]
1. 기술 경로 개요
2. 경로별 비교표
3. 시기별 경로 변화
4. 가장 경쟁이 치열한 경로
5. 비교적 공백이 남은 경로
6. R&D 시사점
```

## 5-D. 경쟁사 비교

### 목적

- 상위 플레이어 간 포지셔닝 비교

### 프롬프트

```text
[Role]
너는 경쟁사 특허 포트폴리오를 비교하는 기술 벤치마킹 분석가다.

[Objective]
입력 특허 리스트에서 상위 출원인들의 Cu paste 기술전략을 비교하라.

[Analysis Rules]
- 상위 출원인 5~10개를 선정하라.
- 각 출원인을 아래 기준으로 비교하라.
  1. 주력 기술경로
  2. 주력 문제 정의
  3. 자주 쓰는 재료/입자 설계
  4. 자주 쓰는 공정
  5. 대표 응용처
  6. 추정 전략
- 추정 전략은 반드시 "추정"으로 표기하라.
- 단순 건수와 질적 우위는 분리해서 서술하라.

[Output Format]
1. 비교 대상 선정 근거
2. 출원인별 요약표
3. 출원인 간 차이점
4. 시장/기술 관점 해석
5. 당사 벤치마킹 포인트
```

## 5-E. 화이트스페이스 탐색

### 목적

- 후속 R&D 기획용 공백 발굴

### 프롬프트

```text
[Role]
너는 특허 분포를 기반으로 R&D 화이트스페이스를 찾는 분석가다.

[Objective]
입력 특허 리스트를 바탕으로 Cu paste 분야에서 상대적으로 비어 있는 기술 영역을 도출하라.

[Analysis Rules]
- 먼저 많이 출원된 영역을 정리하라.
- 그 다음 상대적으로 적은 영역을 도출하라.
- 적은 이유가 검색 누락, 데이터 편향, 실제 기술 공백 중 무엇일 수 있는지 구분하라.
- 화이트스페이스 후보를 최소 5개 제시하라.
- 각 후보별로 R&D 과제화 가능성을 써라.

[Output Format]
1. 밀집 영역
2. 공백 후보 영역
3. 공백 판단 근거
4. 공백이 맞는지 추가 확인에 필요한 데이터
5. 후속 R&D 아이디어
```

## 5-F. 원문 전문 기반 심층 분석

### 목적

- claim, spec, example를 분리해서 진짜 기술 포인트 확인

### 프롬프트

```text
[Role]
너는 특허 명세서와 청구항을 분리 해석하는 시니어 특허기술 분석가다.

[Objective]
입력된 특허 원문을 분석해 청구항의 권리 포인트와 명세서 실시예의 기술 포인트를 구분하라.

[Analysis Rules]
- 아래 항목을 반드시 구분하라.
  1. 청구항에서 넓게 주장하는 요소
  2. 실시예에서 실제로 구현된 요소
  3. 비교예가 있으면 비교예와 실시예 차이
  4. 실제 입증된 성능
  5. 해석상 주의점
- 수치 범위가 있으면 표로 정리하라.
- 증거가 없는 성능은 "입력 텍스트상 근거 부족"으로 적어라.

[Output Format]
1. 특허 개요
2. 청구항 핵심
3. 실시예 핵심
4. 비교예와 차이
5. 실제 기술적 의미
6. 실무 검토 포인트
```

## 5-G. Cu paste 세부 테마 전용 프롬프트

아래는 Cu paste 조사에서 자주 쓰는 세부 테마용 고급 프롬프트다.

### 1. 산화 억제형 Cu paste

```text
입력 특허 리스트에서 산화 억제형 Cu paste 기술만 추려 분석하라.
중점 항목:
- 유기 피복층
- 산화물 제어
- 환원성 분위기
- 저장 안정성
- 소결 전/중/후 산화 억제 메커니즘
결과는 "문제-해결수단-대표 특허-한계" 구조로 정리하라.
```

### 2. 저온 소결형 Cu paste

```text
입력 특허 리스트에서 저온 소결형 Cu paste 기술을 분석하라.
중점 항목:
- 소결 온도
- 활성제
- 입자 구조
- 밀도 및 void 억제
- 접합 강도와 전기저항
단순히 저온이라고 쓰지 말고, 어떤 메커니즘으로 저온화했는지 서술하라.
```

### 3. Ag-coated Cu

```text
입력 특허 리스트에서 Ag-coated Cu 또는 silver-coated copper 계열을 분석하라.
포함:
- 코어와 쉘 조합
- 쉘 형성 방식
- Ag 사용량 저감 논리
```

## 6. 입력 포맷 엔지니어링

로컬 LLM에서는 입력 포맷이 특히 중요하다.

### 6-1. 권장 입력 형식

```text
[Task]
무엇을 분석할지

[Scope]
기간, 국가, 응용처, 비교 대상

[Data Columns]
실제 포함된 컬럼명

[Data]
행 단위 특허 데이터

[Constraints]
근거 없는 추정 금지, 사실/추론 분리 등

[Output]
원하는 출력 형식
```

### 6-2. 행 단위 입력 예시

```text
[Row 12]
국가: JP
출원인: MITSUI MINING & SMELTING CO LTD
출원일: 2018-04-13
제목: SINTERING COMPOSITION BASED ON COPPER PARTICLES
요약: ...
대표청구항: ...

[Row 13]
...
```

이 형식이 좋은 이유:

- 셀 경계가 명확하다
- 모델이 행 간 혼선을 덜 일으킨다
- 이후 결과를 row-based audit로 되돌리기 쉽다

## 7. 출력 포맷 엔지니어링

로컬 LLM에 "좋은 분석"을 기대하면 안 된다.  
"좋은 형식"을 강제해야 "좋은 분석처럼 쓸 수 있는 결과"가 나온다.

### 7-1. 권장 출력 형식

```text
1. 분석 범위
2. 데이터 한계
3. 기초 통계
4. 기술군 분류
5. 대표 특허/대표 기업
6. 해석
7. 추가 필요 데이터
```

### 7-2. 표 강제 문구 예시

```text
반드시 아래 표를 포함하라.
표 1. 국가별 건수
표 2. 출원인별 상위 10개
표 3. 기술군별 대표 특허 유형
표 4. 화이트스페이스 후보
```

### 7-3. 사실/추론 분리 강제 문구

```text
모든 단락 앞에 [Fact] 또는 [Inference]를 붙여라.
근거가 명확하지 않으면 [Inference]로 작성하라.
```

## 8. 환각 억제용 문구

다음 문구는 `gpt-oss-120b`에 매우 중요하다.

```text
입력 데이터에 없는 특허번호, 수치, 기업 전략, 실시예 조건을 만들지 마라.
모호하면 "입력 데이터 기준으로는 확인 불가"라고 적어라.
추정이 필요한 경우 "가능성", "추정", "가설"이라는 단어를 사용하라.
```

추가로 아래 문구도 유용하다.

```text
출력의 완성도보다 근거 충실도를 우선하라.
근거가 없는 문장은 쓰지 마라.
```

## 9. 다단계 분석용 프롬프트 체인

특허 분석은 한 번에 끝내기보다 3단계 체인이 좋다.

### 9-1. Stage 1. 구조 확인

```text
이 데이터의 컬럼 구조를 읽고, 분석 가능한 항목과 분석 불가능한 항목을 구분해라.
```

### 9-2. Stage 2. 행 단위 태깅

```text
각 행에 대해 아래 항목을 태깅하라.
- 직접 관련성
- 기술군
- 응용처
- 핵심 문제
- 핵심 해결수단
- 대표성 여부
```

### 9-3. Stage 3. 종합 보고서

```text
이전에 태깅한 결과만을 근거로 종합 보고서를 작성하라.
태깅에 없는 내용은 새로 만들지 마라.
```

이 방식은 로컬 LLM에서 특히 유용하다.

- 중간 산출물을 검토 가능
- 오류 위치를 추적 가능
- 최종 보고서 품질이 안정적

## 10. 금지해야 할 나쁜 프롬프트

아래 같은 프롬프트는 피해야 한다.

### 나쁜 예 1

```text
이 특허 리스트 보고 중요한 내용 정리해줘.
```

문제:

- 범위 없음
- 출력 형식 없음
- 사실/추론 구분 없음
- 중요성 기준 없음

### 나쁜 예 2

```text
Cu paste 특허를 분석해서 전략을 알려줘.
```

문제:

- 데이터 한계 반영 없음
- 기술 분석과 전략 해석이 섞임
- 환각 가능성 높음

### 나쁜 예 3

```text
이 기업이 왜 이 기술을 개발하는지 설명해줘.
```

문제:

- 기업 의도는 입력 데이터에 없을 수 있음
- 전략 추론을 사실처럼 쓰게 만들 가능성 높음

## 11. 최종 권장 마스터 프롬프트

아래는 Cu paste 특허 조사 전반에 가장 범용적으로 쓸 수 있는 마스터 프롬프트다.

```text
[Role]
너는 금속재료 및 반도체 패키징 특허를 분석하는 시니어 특허기술분석가다.
구리 페이스트, 소결 조성물, 금속 분말, 코어-쉘 입자, 산화 억제, 접합 신뢰성, die attach, 전자부품용 접합 재료 분석에 강하다.

[Objective]
입력된 특허 리스트를 바탕으로 Cu paste 관련 기술을 분석하고, 기초 통계, 기술 분류, 경쟁사 비교, 기술 경로, 화이트스페이스, 후속 조사 포인트를 포함한 구조화된 보고서를 작성하라.

[Input Description]
- 입력은 특허 리스트다.
- 포함 가능 컬럼: 국가코드, 문헌종류, 제목, 요약, 대표청구항, 출원번호, 출원일, 공개번호, 등록번호, 출원인, 발명자, 우선권, 패밀리 ID, 현재권리자 등
- 원문 전문은 없을 수 있다.
- 따라서 실시예 수치나 실제 입증 성능은 제한적으로만 해석해야 한다.

[Analysis Rules]
1. 입력 데이터에 근거한 사실과 해석을 분리하라.
2. 입력 데이터에 없는 정보는 만들지 마라.
3. 출원건수와 기술 우위는 분리해서 해석하라.
4. 제목/요약/대표청구항 기반으로 기술군을 분류하라.
5. 최소 아래 기술군을 사용하라.
   - 입자 설계형
   - 표면/피복형
   - 페이스트 조성형
   - 공정형
   - 응용형
6. 최소 아래 기술 경로를 검토하라.
   - 산화 억제형
   - 소결 촉진형
   - 접합 신뢰성형
   - 비용 절감형
   - 응용 최적화형
7. 상위 출원인을 도출하고, 기술 방향 차이를 비교하라.
8. 데이터 편향, 패밀리 중복, 번역 리스크를 마지막에 명시하라.
9. 추론은 반드시 추론이라고 표시하라.

[Output Format]
1. 분석 범위 및 전제
2. 데이터 한계
3. 기초 통계 요약
4. 국가별 / 연도별 / 출원인별 표
5. 기술 키워드와 기술군 분류
6. 기술 경로 맵
7. 상위 출원인 비교
8. 화이트스페이스 후보
9. 핵심 시사점
10. 추가 확보가 필요한 데이터

[Do Not]
- 존재하지 않는 수치나 특허번호를 만들지 마라.
- 법률 판단처럼 쓰지 마라.
- 실시예 검증이 없는 효과를 확정 표현으로 쓰지 마라.
- 번역문만으로 미세 공정 조건을 단정하지 마라.
```

## 12. 원문 전문 분석용 마스터 프롬프트

```text
[Role]
너는 특허 청구항과 명세서 실시예를 구분해서 해석하는 시니어 특허기술분석가다.

[Objective]
입력된 특허 원문을 분석해 Cu paste 관련 핵심 기술 포인트를 claim, specification, example 기준으로 분리해라.

[Analysis Rules]
1. 청구항에서 넓게 주장하는 요소를 정리하라.
2. 명세서에서 기술되는 핵심 구성과 공정 흐름을 정리하라.
3. 실시예에서 실제로 입증된 수치와 조건을 정리하라.
4. 비교예와의 차이를 정리하라.
5. 실제 입증된 성능인지, 단순 주장인지 구분하라.
6. Cu paste 관점에서 중요한 항목을 반드시 추출하라.
   - 입자 크기
   - 조성 범위
   - 소결 온도
   - 분위기
   - 표면 처리
   - 접합 강도
   - 전기저항
   - 신뢰성

[Output Format]
1. 특허 개요
2. Claim 핵심
3. 명세서 핵심
4. 실시예 핵심
5. 비교예 차이
6. 실제 기술적 의미

```

## 13. 사내 데이터 결합형 마스터 프롬프트

```text
[Role]
너는 특허, 사내 실험, 제품 요구사항을 통합 분석하는 R&D 전략 분석가다.

[Objective]
특허 리스트와 사내 실험 데이터를 함께 분석해 Cu paste 개발 방향을 제안하라.

[Analysis Rules]
1. 외부 특허 기술 흐름을 요약하라.
2. 사내 현재 접근법을 정리하라.
3. 둘 사이의 공정/성능 gap을 찾으라.
4. 특허 충돌 가능성이 높은 방향과 회피 가능성이 높은 방향을 나눠라.
5. 다음 실험 우선순위를 제안하라.

[Output Format]
1. 외부 기술 흐름
2. 사내 현황
3. gap 분석
4. 위험 방향
5. 유망 방향

```

## 14. 결론

로컬 LLM에서 좋은 특허 분석 결과를 얻으려면 "좋은 질문"보다 "좋은 작업 규격"이 중요하다.

특히 `gpt-oss-120b`에는 아래가 핵심이다.

- 역할을 분명하게 줄 것
- 데이터 범위를 먼저 고정할 것
- 사실과 추론을 강제로 분리할 것
- 표와 섹션 구조를 미리 지정할 것
- 장문 특허는 단일 프롬프트보다 단계형 프롬프트 체인으로 처리할 것
- Cu paste는 금속재료 분석과 반도체 패키징 분석을 동시에 걸어둘 것

즉, 프롬프트 엔지니어링의 핵심은 모델을 똑똑하게 만드는 것이 아니라, 모델이 헛똑똑하게 굴 공간을 줄이는 것이다.
